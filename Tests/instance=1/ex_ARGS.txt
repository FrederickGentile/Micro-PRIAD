# This is an example of what your ARGS.txt file should look like. In the ARGS.txt file, all lines that beggins       #
# whit an "# " is a commented line. If there is an argument that you do not want to iitialize and let the program    #
# chose a default value, you can eighter comment the line or earase it completly. To initialize an argument,         #
# you need to write down the name of the argument to initialize and on the right of the name the value, with a       #
# space " " between the name and the value. Here is a description for each argument of what the value given should   #
# look like:                                                                                                         #
#                                                                                                                    #
# - fidelity: you may give fidelity a reel number between 0 and 1 excluding 0                                        #
#                                                                                                                    #
# - seed: you may give seed a positive integer number including 0                                                    #
#                                                                                                                    #
# - PGinstance: you may give instance 1, 2 or 3                                                                      #
#                                                                                                                    #
# - loggingTime: you may give a boolean answer like 0, 1, false or true to activate or desactivate the loggingTime   #
# function that create a file in the $Micro_PRIAD_HOME directory or direcly spesify the directory where you want to  #
# create the timeLog.txt file, you can even force a name to the .txt file (see the diferent example in the           #
# commented line below)                                                                                              #
#                                                                                                                    #
# - halfTrialsReturn: you may give a boolean 0 or 1, this control if a intermediate returen is done before entering  #
#   ElectricSimulation module.                                                                                       #
#                                                                                                                    #
# - N: you may give an integer to change the number of MC trials representing the "truth"                            #
#                                                                                                                    #
# - single_MC_info_return: you may give a boolean answer like 0, 1, false or true to activate or desactivate the     #
# return of information about every Monte Carlo trials. If activated, it return those information in a file in the   #
# $Micro_PRIAD_HOME directory or direcly spesify the directory where you want to  create the .txt file, you can also #
# force a name to the .txt file (see the diferent example in the commented line below)                               #
#                                                                                                                    #
# - continueEval: you may give two arguments that are the two folowing, argument for the continueEval Function are   # 
#   passed via AnyParamForContinueEvalFunction                                                                       # 
#	* the first argument is the directory of the .jl file containing the customize continueEval julia function       #
#	* the second argument is the name of the customized Julia function                                               #
#    + you may use the basicContinueEval that simply return true (TO USE if you are not inerested in the             #
#      intermidite return).                                                                                          #
#    + you can also use a builted in continueEval (printInterReturnConinueEval) that communicate via a .txt file     #
#      with an external algorithm, or simply to see the value of the BB evolv during an evaluation. To call this     #
#      continueEval, you must give one or two additionnal argument the first one corresponding to the .txt file      #
#      where the intermediate value will be printed and the second one creesponding to the .txt file where the       #
#      algorithme will write true or false to tell the BB to stop the evaluation or continue the evaluation.         #
#      (see examples in the commented lines below)                                                                   #                   
#                                                                                                                    #
# - AnyParamForContinueEvalFunction: you may give as many arguments as you want that your continueEval function      #
#   will handles it self. Some exemple for the builted in continueEvalfunction are shown in the commented line below #

# - eta: you may give a integer between 1 and 10000, this argument contrôl how often the                             #
